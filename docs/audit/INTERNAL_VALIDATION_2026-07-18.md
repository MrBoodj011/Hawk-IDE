# Hawk internal validation — 2026-07-18

This report records reproducible engineering evidence. It is not an external
pentest certificate and does not replace independent assessment.

## Semantic index beta

Each repository was measured in a fresh Node.js process on Windows x64 with
Node.js 24.18.0. The index used a persistent incremental AST/type-aware store
with a ceiling of 8,000 files, 10,000 chunks, and 48 MiB of source.

| Project | Indexed | Cold | Warm | Incremental | Search samples | RSS |
| --- | ---: | ---: | ---: | ---: | ---: | ---: |
| Microsoft TypeScript | 1,968 files / 9,999 chunks | 5.25 s | 0.56 s | 0.54 s | 40–51 ms | 868 MiB |
| Microsoft Visual Studio Code | 1,680 files / 9,987 chunks | 5.70 s | 0.72 s | 0.53 s | 45–47 ms | 925 MiB |
| OWASP Juice Shop | 966 files / 2,906 chunks | 1.93 s | 0.17 s | 0.11 s | 11–13 ms | 409 MiB |

The TypeScript and VS Code runs reached the bounded chunk ceiling as expected.
Queries still returned relevant code for language services, type checking,
module resolution, debug adapters, inline completion, and extension-host
lifecycle. Juice Shop was indexed without truncation.

The beta exposed and fixed two robustness defects before release:

- deeply nested TypeScript syntax could overflow a recursive AST traversal;
- compiler parser-crash reproducers could throw inside the TypeScript parser.

Hawk now uses a bounded iterative traversal and a safe structural fallback.

Re-run all repositories:

```powershell
npm run benchmark:beta-index
```

Run one repository in an isolated process:

```powershell
node --expose-gc node_modules/tsx/dist/cli.mjs scripts/beta-index-benchmark.ts --clone --project visual-studio-code
```

## Real private updater test

The updater test used the private GitHub release feed and exercised the
`0.2.0 -> 0.2.1` upgrade path without launching the installer.

- Asset: `HawkSetup-windows-x64-0.2.1.exe`
- Size: `256,373,693` bytes
- SHA-256:
  `68913a361e6c076fdc28cd6410b79a91363d05d774c9ed669591c83678dd72e5`
- Release metadata, asset size, checksum manifest, downloaded bytes, and PE
  header: verified
- Authenticode: `NotSigned`

The unsigned status is expected for the historical local-built release and is
the reason the production workflow refuses to publish a new Windows release
until a trusted PFX or Microsoft Artifact Signing profile is configured.

Re-run:

```powershell
npm run test:update-real
```

Add `--require-signature` to the underlying script for a signed release gate.

## Automated checks

The release candidate must pass:

- TypeScript type checking and Biome lint;
- unit and integration tests, including deep-AST, multiline next-edit,
  authenticated daemon protocol, isolated apply/revert, and durable
  pause/resume recovery;
- production build, extension check/package, integration packaging, and Burp
  companion build;
- dependency vulnerability audit;
- PowerShell signing-script parse validation and workflow YAML validation.

## Remaining independent evidence

Before calling Hawk publicly audited, an independent assessor must execute
`HAWK_EXTERNAL_PENTEST_SCOPE.md` against the signed candidate. Real users must
also complete `BETA_TEST_PLAN.md` on representative repositories. These are
external outcomes and cannot be honestly generated by Hawk's own source tree.
